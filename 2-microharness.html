// 01 · 背景

为什么要自己搭，而不是直接用 DeepAgents？

DeepAgents、Claude Code 这类工具帮你打包好了一切——开箱即用，但你看不到里面在干什么。

自己搭的价值在于：每一层你都能看见、修改、替换。这是学习 Harness 原理最直接的方式，也是日后为你的供应链平台定制 Harness 的基础。

这个 demo 刻意保持最小——9 个文件，不到 400 行代码，完整覆盖 Harness 的5个核心概念，包括跨会话的长期记忆持久化，以及支持八个模型 Provider 的统一配置层。

// 02 · 快速开始

三步跑起来

bash # 1. 下载zip文件解压，或者git clone后进入目录 unzip microharness.zip && cd microharness # 2. 安装核心依赖 pip install -r requirements.txt # 3. 非 Anthropic provider 额外安装 pip install langchain-openai # DeepSeek / Kimi / Qwen / GLM / MiniMax / OpenAI / MiMo # 4. 编辑 .env，配置 Provider 和 API Key # 5. 运行 python harness.py

启动后输入任务，例如：

example Task: Write a Python script that prints Fibonacci numbers up to 100, then run it

Harness 会自动：写文件 → 守卫拦截请求你确认 → 你批准 → 执行 → 读取结果 → 返回总结。

// 05 · 执行流程

LangGraph 图的运转方式

整个 Harness 是一个 LangGraph StateGraph，节点之间通过条件路由连接：

👤
User Input
任务输入 + 初始化 State（step_count=0）
🤖
agent_node
注入系统提示（含长期记忆）→ 模型推理 → step_count +1
🛡️
guard_node
检查工具调用 → 写操作拦截 → 等待人工 yes/no
🛠️
tool_node
执行工具（write_file / run_python / list_files）→ 结果写回 State
🔁
→ 回到 agent_node
继续推理，直到任务完成或达到 MAX_STEPS
🧠
memory.extract_and_save()
会话结束 → Haiku 提炼摘要 → 写入 memory.json → 下次启动注入提示
// 06 · 安全守卫详解

哪些操作会被拦截？

守卫层对不同工具有不同策略——这是 Harness 和普通 Agent 最关键的区别：

list_files → ✅ 自动放行 — 只读，无副作用
write_file → ⏸ 强制确认 — 写操作，默认拦截，等待 yes/no
run_python → ⚠ 关键词检测 — 含 rm/DELETE/rmtree 等则拦截

拒绝后 Harness 路由到 END，不会继续执行任何工具调用。

// 06.5 · 多模型支持

切换 Provider 只需改 .env

所有 Provider 的切换逻辑集中在 config.py，用户不需要看代码，只改 .env 的三行：

.env — 以 DeepSeek 为例 PROVIDER=deepseek OPENAI_COMPATIBLE_API_KEY=your_deepseek_key MAIN_MODEL=deepseek-chat
Provider PROVIDER= 常用 MAIN_MODEL Base URL（内置）
Anthropic anthropic claude-sonnet-4-20250514 —
DeepSeek deepseek deepseek-chat api.deepseek.com
Kimi kimi moonshot-v1-8k api.moonshot.cn/v1
Qwen qwen qwen-plus dashscope.aliyuncs.com/...
GLM glm glm-4-air open.bigmodel.cn/...
MiniMax minimax abab6.5s-chat api.minimax.chat/v1
OpenAI openai gpt-4o api.openai.com/v1
自定义 任意值 — OPENAI_COMPATIBLE_BASE_URL 填入

非 Anthropic provider 需额外安装：pip install langchain-openai
